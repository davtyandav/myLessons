Inversion of control.

ответственным для создания и управления объектов является спринг контейнер .
это то контейнер где будут находиться наши объекты и мы будем получать их.
контейнер читает конфиг файл и в контейнере будут создаваться объекты который мы опишем в конфиг файле

ioc- инверсия управления
создания и управления объектами

Di - Dependency injection
внедрения зависимости

есть 3 способа создания конфигурации
 1. XML file
 2. Annotation + XMl file
 3. Java code

 Bean -  это просто тот объект который создается и управляется с помощью Spring контейнера
 ApplicationContext представляет собой Spring container поэтому для получения bina нужно создать ApplicationContext.
