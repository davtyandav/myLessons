В Java для управления состоянием потоков используются различные методы и константы из класса Thread и интерфейса Runnable.
 Основные состояния потоков в Java:

    NEW (новый): Поток был создан, но его метод start() еще не был вызван.

    RUNNABLE (выполняемый): Поток выполняется или ожидает выполнения в планировщике потоков.
     Это состояние означает, что поток может выполняться или ожидает своей очереди на выполнение.

    BLOCKED (заблокированный): Поток ожидает блокировки для доступа к ресурсу, который уже заблокирован другим потоком.
     Например, это может быть ожидание доступа к критическому участку кода, защищенному synchronized.

    WAITING (ожидающий): Поток находится в состоянии ожидания, ожидая события, которое может произойти для его пробуждения.
     Это может быть вызвано методами wait(), join() или park().

    TIMED_WAITING (ожидающий с тайм-аутом): Поток находится в состоянии ожидания с тайм-аутом.
     Например, это состояние может быть вызвано методами sleep(), wait(timeout) или join(timeout).

    TERMINATED (завершенный): Поток завершил выполнение.
     Это состояние наступает, когда метод run() завершает свою работу или вызывается метод stop().
